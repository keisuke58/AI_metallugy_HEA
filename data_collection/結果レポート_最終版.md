# 📊 HEA弾性率予測プロジェクト 結果レポート

**作成日**: 2026年1月20日  
**プロジェクト**: 高エントロピー合金（HEA）の弾性率予測モデル開発

---

## 📋 エグゼクティブサマリー

本プロジェクトでは、機械学習を用いて高エントロピー合金（HEA）の弾性率（Young's Modulus）を予測するモデルを開発しました。複数のデータソースから322個のデータポイントを収集し、8種類の機械学習モデルを訓練・評価しました。

### 主要成果

- ✅ **データ収集**: 322個の弾性率データを統合
- ✅ **特徴量エンジニアリング**: 29個の特徴量を生成
- ✅ **モデル開発**: 8種類のモデルを実装・評価
- ✅ **最良性能**: R² = 0.5945（SVR最適化版）、R² = 0.6744（Gradient Boosting、実測データのみ）
- ✅ **可視化**: 21個の図表を作成

---

## 1. データ収集結果

### 1.1 データソース

| データソース | データ数 | 説明 |
|------------|---------|------|

| **Gorsse Dataset** | 211個 | 文献データベース（統合後） |
| **最新研究データ** | 4個 | 2024-2025年の最新論文から抽出 |
| **合計** | **322個** | 最終統合データ数 |

### 1.2 データクリーニング

- **重複除去**: 36個
- **NaN除去**: 124個
- **最終データ数**: 322個
- **弾性率範囲**: 27.00 - 466.00 GPa
- **平均弾性率**: 165.40 GPa
- **標準偏差**: 74.58 GPa

### 1.3 データ特性

- **生体医学用途データ**: 30-90 GPa範囲に30個の合金データを含む
- **データ品質**: 良好（欠損率は許容範囲内）
- **データ分布**: 正規分布に近い分布を示す

## 2. 特徴量エンジニアリング

### 2.1 特徴量の種類

**最終特徴量数**: 29個

#### 材料記述子（Material Descriptors）
- 原子半径関連: 平均、標準偏差、最大、最小、差（delta_r）
- 電気陰性度関連: 平均、標準偏差、差（delta_chi）
- 価電子濃度（VEC）
- 混合エントロピー
- 元素数

#### 組成特徴量（Composition Features）
- 各元素の組成比: Ti, Zr, Hf, Nb, Ta, V, Cr, Mo, W, Fe, Co, Ni, Cu, Al, Mn, Si, Sn

#### 物理特性
- 密度（density）
- 混合エンタルピー
- 価電子数

### 2.2 特徴量重要度（Random Forest）

**上位5つの重要特徴量**:

1. **comp_Ti** (16.6%): Ti組成比
2. **mean_electronegativity** (8.8%): 平均電気陰性度
3. **vec** (8.0%): 価電子濃度
4. **comp_Co** (6.8%): Co組成比
5. **density** (6.4%): 密度

**重要な発見**: Ti組成比が最も重要な予測因子であることが判明しました。

## 4. モデル性能の分析

### 4.1 成功した点

1. **複数モデルの実装**
   - 8種類の異なる機械学習モデルを実装
   - 線形モデルから非線形モデルまで幅広く評価

2. **最適化による性能向上**
   - SVR: R² 0.36 → 0.59（67%向上）
   - Gradient Boosting: R² 0.49 → 0.67（37%向上）

3. **特徴量エンジニアリングの成功**
   - 29個の特徴量を生成
   - Ti組成比が最も重要な因子であることを発見

4. **実用的な予測精度**
   - R² = 0.59-0.67は実用的な予測性能を示す
   - RMSE = 36-50 GPaは許容範囲内

### 4.2 改善が必要な点

1. **過学習の抑制**
   - Random Forest: Train R² (0.88) >> Test R² (0.57)
   - より強い正則化が必要

2. **データ数の増加**
   - 現在322個（目標の64-80%）
   - より多くのデータで性能向上が期待される

3. **Polynomial Regressionの改善**
   - 過学習が激しい（R² = -1.94）
   - より強い正則化が必要

## 6. 重要な発見と洞察

### 6.1 データに関する発見

1. **データ品質**
   - 322個のデータで基本的なモデル訓練は可能
   - 実測データのみ（109個）でも良好な性能を達成

2. **データ分布**
   - 弾性率範囲: 27-466 GPa
   - 生体医学用途（30-90 GPa）のデータも含まれる

### 6.2 特徴量に関する発見

1. **最重要特徴量**
   - Ti組成比（16.6%）が最も重要
   - 電気陰性度と価電子濃度も重要

2. **特徴量エンジニアリングの効果**
   - 29個の特徴量が予測に寄与
   - 材料記述子が特に有効

### 6.3 モデルに関する発見

1. **最適なモデルタイプ**
   - 非線形モデル（SVR、Gradient Boosting、Random Forest）が良好
   - 線形モデルは中程度の性能

2. **最適化の重要性**
   - ハイパーパラメータ最適化で大幅な性能向上
   - SVR: 67%の改善、Gradient Boosting: 37%の改善

3. **データセットの影響**
   - 実測データのみでより高い性能（R² = 0.67）
   - 計算データを含めると性能が低下する傾向

## 7. 結論と今後の展望

### 7.1 結論

本プロジェクトは、機械学習を用いたHEA弾性率予測モデルの開発に成功しました。

**主要な成果**:
- ✅ 322個のデータを収集・統合
- ✅ 29個の特徴量を生成
- ✅ 8種類のモデルを実装・評価
- ✅ 最良性能: R² = 0.5945（SVR最適化版）、R² = 0.6744（Gradient Boosting、実測データのみ）
- ✅ 21個の可視化図表を作成

**実用的な価値**:
- 新規HEA合金の弾性率を予測可能
- 材料設計の効率化に貢献
- 実験コストの削減

### 7.2 今後の改善案

#### 短期的改善（1-3ヶ月）

1. **データ収集の拡大**
   - 目標: 500-600個のデータポイント
   - より多様な組成のデータを追加

2. **特徴量の拡充**
   - 相情報を特徴量として追加
   - 処理条件（熱処理、冷間加工など）を追加
   - 第一原理計算による特徴量を追加

3. **モデルの改善**
   - より強い正則化で過学習を抑制
   - アンサンブル手法の最適化
   - 深層学習モデルの検討

#### 中長期的改善（3-12ヶ月）

1. **転移学習の適用**
   - 類似材料のデータを活用
   - ドメイン適応手法の検討

2. **不確実性の定量化**
   - 予測区間の信頼性向上
   - ベイジアン手法の検討

3. **マルチタスク学習**
   - 弾性率だけでなく、降伏強度、硬度も同時に予測
   - 材料特性の統合予測

4. **アクティブラーニング**
   - 予測不確実性が高い組成を優先的に実験
   - 効率的なデータ収集戦略

## 8. 技術的詳細

### 8.1 使用した技術スタック

- **プログラミング言語**: Python 3.x
- **機械学習ライブラリ**: scikit-learn, TensorFlow/Keras
- **データ処理**: pandas, numpy
- **可視化**: matplotlib, seaborn
- **最適化**: GridSearchCV, RandomSearchCV

### 8.2 モデルアーキテクチャ

#### Random Forest
- n_estimators: 100-500
- max_depth: 5-20
- min_samples_split: 2-10

#### SVR（最適化版）
- kernel: RBF
- C: 最適化済み
- gamma: 最適化済み
- epsilon: 最適化済み

#### Gradient Boosting
- n_estimators: 200-800
- max_depth: 3-12
- learning_rate: 0.01-0.1

#### MLFFNN
- 層数: 3-5層
- 活性化関数: ReLU
- 最適化器: Adam

### 8.3 評価方法

- **データ分割**: Train/Test = 80/20
- **交差検証**: 5-fold CV
- **評価指標**: R², RMSE, MAE
- **過学習評価**: Train R² vs Test R²
