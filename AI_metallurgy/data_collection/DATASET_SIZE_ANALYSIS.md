## ⚠️ 注意点

### 1. データの質が重要

- **量だけでなく質も重要**: 多様性のあるデータが必要
- **バランス**: 目標範囲（30-90 GPa）内のデータも増やす
- **ノイズ**: 低品質なデータを追加すると性能が低下する可能性

### 2. 過学習の抑制

現在、Random Forestで過学習が発生しています：
- Train R² (0.88) >> Test R² (0.57)
- データを増やすことで過学習は減少しますが、同時に正則化も重要

### 3. 収穫逓減の法則

- データを増やすほど改善率は減少
- 580サンプル → 1,073サンプルへの改善は、322 → 580への改善より小さい可能性
