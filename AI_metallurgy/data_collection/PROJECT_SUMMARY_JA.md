# 高エントロピー・多主成分合金の弾性率予測プロジェクト 日本語サマリー

作成日: 2026-01-23  
プロジェクト: AI\_metallurgy（HEA/MPEA 弾性率予測）  

---

## 1. プロジェクト概要（1ページ以内）

### 1.1 背景と目的

金属インプラント（人工股関節・人工膝関節など）では、**骨と金属の弾性率ミスマッチ**が大きな課題です。  
ヒト皮質骨のヤング率はおおよそ 10–30 GPa 程度であるのに対し、一般的な金属材料（Ti-6Al-4V, Co–Cr 合金, ステンレス鋼など）は 100–200 GPa 以上であり、  
これが **ストレスシールド** や骨吸収、長期的なインプラントゆるみの原因となります。

本プロジェクトでは、**高エントロピー合金（HEA）および多主成分合金（MPEA）** を対象に、

- ヤング率（弾性率）データを大規模に統合・整理し、  
- 機械学習モデルによって合金組成から弾性率を予測し、  
- 特に **30–90 GPa 程度の「骨に近い」弾性率** を持つ合金候補を探索すること

を目的としました。

### 1.2 使用データの概要

最終的な統合データセットは、実験および計算を含む複数の公開データベースから構築され、

- **総サンプル数**: 5,339（重複・欠損処理後）  
  - 実験データ: 約 335 件（約 6.3%）  
  - 計算データ（DFT など）: 約 5,004 件（約 93.7%）  
- ヤング率の範囲: 約 2.7–621 GPa  
- 平均ヤング率: 約 139 GPa  

といった特性を持ちます。

モデル開発の初期段階では、とくに

- DOE/OSTI, Gorsse データベースなどからなる **実験中心の 322 サンプル**

を用いて詳細なモデル評価と解釈を行いました（弾性率 27–466 GPa、うち 30–90 GPa の骨近傍データが約 30 件）。

### 1.3 使用したモデルと主な成果

29 個の特徴量（組成比・原子半径・電気陰性度・VEC・密度など）を用い、以下のような機械学習モデルを比較・最適化しました。

- 線形モデル: Linear, Ridge, Lasso  
- 非線形モデル: Polynomial, KNN, Random Forest, Support Vector Regression (SVR), Gradient Boosting, Stacking Ensemble  
- ニューラルネットワーク: 多層フィードフォワード NN (MLFFNN)  

その結果：

- **全データセット（322 サンプル）に対する最適化 SVR**  
  - Test \(R^2 \approx 0.59\)  
  - RMSE \(\approx 36\) GPa  
- **実測データのみ（約 109 サンプル）に対する Gradient Boosting**  
  - Test \(R^2 \approx 0.67\)  
  - RMSE \(\approx 50\) GPa  

といった性能を達成し、**中程度〜良好な予測精度** を実現しました。

## 2. データと手法（0.5–1ページ）

### 2.1 データソースと統合フロー

本プロジェクトで利用した主なデータソースは以下の通りです（詳細は `DATASET_DOCUMENTATION.md` 参照）。

- DOE/OSTI 高エントロピー合金データセット（実験）  
- Gorsse HEA 機械特性データベース（実験）  
- 最新 HEA 実験データ（Ti–Zr–Nb 系, Ti–Zr–Hf–Nb–Ta 系など）  
- MPEA ナノインデンテーションデータ（実験 + 計算）  
- Refractory HEA 弾性定数データセット（DFT 計算）  
- Materials Project 弾性テンソルデータ（DFT 計算）  

データ統合の流れ（テキストによる図解イメージ）は以下の通りです。

1. **各ソースからの取り込み**  
   - 生データ（csv, xlsx など）を読み込み、ヤング率列を `elastic_modulus`（単位 GPa）に統一。  
2. **前処理・クリーニング**  
   - `elastic_modulus <= 0` や欠損値を除去。  
   - 合金名・組成・弾性率がほぼ同一な重複行を削除。  
3. **統合テーブルへの集約**  
   - 必須列（合金名, elastic_modulus, source）が揃っている行のみを残して結合。  
4. **検証スクリプトによる品質チェック**  
   - 欠損率、外れ値、重複、ソース別件数などを自動レポート化。  

こうして構築した統合データセットは、`final_data/unified_dataset_latest.csv` に保存されており、  
機械学習モデルの学習および評価のベースとなっています。

### 2.2 特徴量エンジニアリングの要点

最終的に使用した特徴量は **29 個** で、大きく以下のグループに分かれます。

- **材料記述子 (material descriptors)**  
  - 原子半径関連: 平均、標準偏差、最大、最小、差  
  - 電気陰性度関連: 平均、標準偏差、差  
  - 価電子濃度 (VEC)、混合エントロピー、元素数 など  

- **組成特徴量 (composition features)**  
  - Ti, Zr, Hf, Nb, Ta, V, Cr, Mo, W, Fe, Co, Ni, Cu, Al, Mn, Si, Sn  
    の **17 元素の組成比**  

- **物理特性**  
  - 密度（利用可能なデータに対して）  
  - 一部データセット由来の既存特徴量（例: 既存の記述子カラム）  

これらの特徴量は、合金の組成情報と元素物性表（原子半径・電気陰性度など）から計算され、  
機械学習モデル入力として標準化・スケーリングを施しています。

## 3. 主な結果（1ページ）

### 3.1 モデル性能のまとめ

実験中心の 322 サンプルを用いたベースライン比較では、

- **Random Forest** がもっとも良好な初期性能を示し、  
  - Test \(R^2 \approx 0.57\)、RMSE \(\approx 37.5\) GPa  
- 線形系モデル（Linear, Ridge, Lasso）は  
  - \(R^2 \approx 0.35–0.37\) と中程度  
- Polynomial Regression は強い過学習により  
  - Test \(R^2 < 0\)（不適切）  
- MLFFNN は  
  - \(R^2 \approx 0.49\) 程度の性能  

という結果でした。

その後のハイパーパラメータ最適化により、

- **SVR（RBF カーネル）最適化版**  
  - 全 322 サンプルに対して  
  - Test \(R^2 \approx 0.59\)、RMSE \(\approx 36\) GPa  
- **Gradient Boosting 最適化版（実測データのみ）**  
  - 約 109 サンプルの実験データ subset に対して  
  - Test \(R^2 \approx 0.67\)、RMSE \(\approx 50\) GPa、MAE \(\approx 37\) GPa  

を達成しました。  
この \(R^2 \approx 0.6–0.7\) 程度の精度は、データのばらつき・混合ソースを考慮すると **実用的なレベル** と評価できます。

### 3.2 重要な特徴量（feature importance）のポイント

Random Forest や Gradient Boosting による特徴重要度解析から、特に以下の特徴量が重要であることが分かりました。

1. **Ti 組成比 (`comp_Ti`)**  
   - 単独で約 16–17% 程度の寄与。  
   - 生体適合 HEA/MPEA では Ti が主成分となることが多く、弾性率を下げる方向に効く場合が多い。  
2. **平均電気陰性度**  
   - 結合の性質や相安定性に関係し、弾性率に系統的な影響を与える。  
3. **価電子濃度 (VEC)**  
   - BCC/FCC などの相選択や結合状態と関連し、剛性の変化に寄与。  
4. **Co 組成比 (`comp_Co`)**  
   - Co を多く含む合金では弾性率が高い傾向があり、剛性増大要因として効く。  
5. **密度 (density)**  
   - 重い難融金属（W, Mo, Ta など）を多く含む合金は高密度・高弾性率となることが多い。  

総じて、**Ti リッチな組成** や **電気陰性度・VEC の制御** により、弾性率をある程度設計可能であることが示唆されました。

### 3.3 生体用弾性率レンジ（30–90 GPa）の合金候補

322 サンプルの実験データのうち、

- **30–90 GPa の「骨近傍」レンジに入る合金** が約 30 件存在し、  
- これらの多くは Ti–Zr–Nb–Ta–Hf 系などの **Ti 基高エントロピー/多主成分合金** です。  

最良モデル（SVR, Gradient Boosting など）は、これらの合金の弾性率を数十 GPa 程度の誤差で再現できており、  
完全に精密な設計にはまだ課題が残るものの、**探索空間を大きく絞り込むツール** として有用であると考えられます。

## 4. 考察と今後の課題（0.5–1ページ）

### 4.1 実測データ vs 計算データの扱い

統合データセットは、

- 実験データ: 約 6.3%  
- 計算データ（DFT など）: 約 93.7%  

という強いアンバランスを持っています。  
計算データを含めることで組成空間を大きくカバーできる一方、

- DFT による弾性率は **実験値より高めに見積もられる** 傾向がある  
- 温度条件・組織状態が実験と計算で大きく異なる  

といったバイアスも存在します。  
そのため本プロジェクトでは、

1. 学習には統合データセットを活用しつつ、  
2. 評価や解釈は **実測データ subset（322 or 335 サンプル）での性能** を重視する  

という方針を採用しました。

### 4.2 データ・モデルの限界

主な限界要因は以下の通りです。

- **実測データ数の不足**:  
  - 特定の組成領域（とくに生体用途 HEA/MPEA）ではデータがまだ少なく、  
    予測不確実性が高くなりがち。  
- **加工条件・組織情報の欠如**:  
  - 熱処理や冷間加工などの条件が統一されておらず、多くは記録もされていない。  
- **モデルの解釈性と複雑さのトレードオフ**:  
  - SVR や Gradient Boosting は高精度だが、線形モデルに比べると直感的な解釈が難しい。  

それでもなお、本プロジェクトで構築したデータセットとモデルは、  
**「どのあたりの組成が低弾性率候補になりうるか」を素早くスクリーニングするツール** として十分有用です。

### 4.3 今後の展望

今後の発展方向として、以下を想定しています。

- **データ拡張と精緻化**  
  - 生体用途 HEA/MPEA の実測データを追加収集し、5,339 サンプルに対する実験比率を高める。  
  - 加工条件・相情報などのメタデータを統合し、多変量的な解析を可能にする。  

- **高度なモデルの導入**  
  - 本プロジェクト内の別フォルダで進めている GNN や FNO（Fourier Neural Operator）などを  
    弾性率予測にも適用する。  
  - 弾性率だけでなく、強度・硬度などを同時予測する **マルチタスク学習** への拡張。  

- **不確実性評価とアクティブラーニング**  
  - 予測不確実性を定量化し、「どの組成を優先的に実験すべきか」を提案するフレームワークの構築。  

- **逆設計（インバースデザイン）**  
  - 学習済みモデルと最適化アルゴリズムを組み合わせ、  
    `30–90 GPa` の弾性率かつ一定以上の強度を満たす合金組成候補を自動設計する。
